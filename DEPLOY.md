# 🏢 แผนกระจายให้คนในองค์กร (ASEFA)

วิธีทำให้พนักงานโหลดไปติดตั้งบนเครื่องตัวเองได้ง่ายๆ

---

## ภาพรวม 3 ส่วน

```
[ IT/Admin ทำครั้งเดียว ]              [ พนักงานแต่ละคน ]
  1. ตั้งค่า OAuth ฝั่งองค์กร            1. โหลด ASEFA-Mail-Setup.exe
  2. Build ตัวติดตั้ง (.exe)      ──►    2. ดับเบิลคลิกติดตั้ง
  3. วางไฟล์ให้ดาวน์โหลด                 3. Login บัญชี Zoho ตัวเอง → ใช้ได้
```

---

## ส่วนที่ 1 — IT/Admin: Build ตัวติดตั้ง (ทำครั้งเดียว)

บนเครื่องที่มี Python:

```bat
build_app.bat        REM → dist\ASEFAMail.exe   (แอปหลัก พร้อมโลโก้ ASEFA)
build_exe.bat        REM → dist\ZohoMailNotifier.exe  (ตัวแจ้งเตือน optional)
```

แล้วแพ็กเป็นตัวติดตั้งเดียว (มี shortcut/Start Menu เหมือนลง Word):
1. ติดตั้ง **Inno Setup** (ฟรี): https://jrsoftware.org/isdl.php
2. เปิด `installer.iss` → **Compile**
3. ได้ `Output\ASEFA-Mail-Setup.exe`  ← ไฟล์เดียวแจกได้

---

## ส่วนที่ 2 — วางไฟล์ให้พนักงานโหลด

เลือกที่วางไฟล์ `ASEFA-Mail-Setup.exe`:
- 📁 Shared drive / ไฟล์เซิร์ฟเวอร์ภายใน
- 🌐 SharePoint / Intranet
- 📧 ส่งลิงก์ทางอีเมล

> ขนาดไฟล์ ~30-50 MB (รวม Python ในตัว ผู้ใช้ไม่ต้องลงอะไรเพิ่ม)

---

## ส่วนที่ 3 — พนักงาน: ติดตั้ง + ใช้งาน

1. โหลด `ASEFA-Mail-Setup.exe` → ดับเบิลคลิก → Next → Install
2. เปิดแอป (ไอคอน ASEFA บนเดสก์ท็อป)
3. **เข้าสู่ระบบด้วยบัญชี Zoho ของตัวเอง** → ใช้ได้เลย

---

## ⚠️ จุดสำคัญที่สุด — พนักงานจะ "ยืนยันตัวตน" อย่างไร?

แต่ละคนต้องมี token ของอีเมลตัวเอง มี 3 วิธี — **ต้องเลือก 1**:

### วิธี A — แต่ละคนสร้าง Self Client เอง  ❌ ไม่เหมาะองค์กร
พนักงานต้องเข้า api-console.zoho.com สร้าง client เอง — ยุ่งยากเกินไปสำหรับคนทั่วไป
(นี่คือวิธีที่โค้ดตอนนี้ใช้)

### วิธี B — องค์กรลงทะเบียน OAuth App กลาง + พนักงาน Login ด้วยบัญชีตัวเอง  ✅ แนะนำ
- **Admin ทำครั้งเดียว:** ลงทะเบียน "Server-based Application" ที่ api-console.zoho.com
  ตั้ง redirect URI = `http://localhost:8765/callback` → ได้ Client ID/Secret กลาง
  (ฝังไว้ในแอป หรือแจกไฟล์ config)
- **พนักงาน:** กดปุ่ม "เข้าสู่ระบบด้วย Zoho" → เบราว์เซอร์เปิดหน้า login Zoho →
  ล็อกอินบัญชีตัวเอง → อนุญาต → แอปรับ token อัตโนมัติ (ไม่ต้อง copy code)
- **ข้อดี:** พนักงานแค่ล็อกอินเหมือนเว็บทั่วไป ไม่ต้องสร้าง client/กรอก ID
- **ต้องสร้างเพิ่ม:** ระบบ login แบบ browser callback (ผมทำให้ได้)

### วิธี C — IT เตรียม token ให้แต่ละคนล่วงหน้า
IT รัน OAuth ให้ทีละคน แล้วแจกไฟล์ `.env` พร้อม token ไปวางในเครื่อง —
ควบคุมได้เต็มที่ แต่ IT ต้องทำเองทุกคน

---

## Checklist สำหรับ Admin

- [ ] เลือกวิธียืนยันตัวตน (A / B / C) — ดูด้านบน
- [ ] (ถ้า B) ลงทะเบียน OAuth App กลาง + ตั้ง redirect URI
- [ ] `build_app.bat` → ได้ ASEFAMail.exe
- [ ] Inno Setup compile → ASEFA-Mail-Setup.exe
- [ ] ทดสอบติดตั้งบนเครื่องเปล่า (ไม่มี Python) 1 เครื่อง
- [ ] วางไฟล์ให้พนักงานโหลด + เขียนคู่มือสั้นๆ
- [ ] (แนะนำ) revoke Client Secret เดิมที่เคยใช้ทดสอบ แล้วออกใหม่

---

## หมายเหตุความปลอดภัย

- วิธี B: Client Secret ฝังในแอป = พนักงานในองค์กรเข้าถึงได้ (ยอมรับได้สำหรับ internal tool)
  ถ้าต้องการเข้มกว่านี้ ใช้วิธี C หรือทำ proxy server กลาง
- token ของแต่ละคนเก็บใน `.env` บนเครื่องตัวเอง ไม่ถูกแชร์
- `.env` / `.token_cache.json` ไม่ถูก commit ขึ้น git
